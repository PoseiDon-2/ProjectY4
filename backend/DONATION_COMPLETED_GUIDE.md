# 🎉 คู่มือการจัดการเมื่อบริจาคครบจำนวน

## สิ่งที่ระบบทำอัตโนมัติเมื่อครบจำนวน

### 1. **เปลี่ยนสถานะคำขอ**
- สถานะเปลี่ยนจาก `APPROVED` เป็น `COMPLETED`
- คำขอจะไม่แสดงในรายการที่เปิดรับบริจาคแล้ว

### 2. **การแจ้งเตือน (Notifications)**
- **Organizer** จะได้รับแจ้งเตือนว่า "คำขอบริจาคครบจำนวนเป้าหมายแล้ว"
- **ผู้บริจาคคนสุดท้าย** จะได้รับแจ้งเตือนขอบคุณ

### 3. **ป้องกันการบริจาคเพิ่ม**
- ระบบจะปฏิเสธสลิปใหม่ที่ส่งมาหลังจากครบจำนวน
- แสดงข้อความว่า "คำขอนี้บริจาคครบจำนวนเป้าหมายแล้ว"

## สิ่งที่ Organizer ควรทำหลังจากครบจำนวน

### ✅ **ขั้นตอนที่แนะนำ:**

1. **ตรวจสอบยอดเงิน**
   - ตรวจสอบใน Dashboard ว่ายอดเงินถูกต้อง
   - ตรวจสอบสลิปทั้งหมดที่อนุมัติแล้ว

2. **สร้าง Story ขอบคุณ**
   - สร้าง Story ประเภท "thank_you" หรือ "completion"
   - แสดงภาพความคืบหน้าและผลลัพธ์
   - ขอบคุณผู้บริจาคทุกคน

3. **อัปเดตความคืบหน้า**
   - แชร์ภาพ/วิดีโอการใช้งานเงินบริจาค
   - แสดงผลลัพธ์ที่เกิดขึ้น

4. **ปิดคำขอ (ถ้าต้องการ)**
   - คำขอจะถูกปิดอัตโนมัติเมื่อครบจำนวน
   - แต่สามารถสร้าง Story ต่อได้

5. **จัดการเงินบริจาค**
   - ถอนเงินจากบัญชี
   - ใช้เงินตามวัตถุประสงค์ที่ระบุ
   - บันทึกหลักฐานการใช้งาน

## Frontend Features ที่ควรมี

### 1. **แสดงสถานะ "ครบจำนวนแล้ว"**
```tsx
{request.status === 'COMPLETED' && (
  <Badge className="bg-green-500">✅ บริจาคครบจำนวนแล้ว</Badge>
)}
```

### 2. **ปิดการรับบริจาค**
- ซ่อนปุ่ม "บริจาค" เมื่อ status เป็น COMPLETED
- แสดงข้อความ "คำขอนี้บริจาคครบจำนวนแล้ว"

### 3. **แสดงความสำเร็จ**
- แสดง Animation หรือ Badge เมื่อครบจำนวน
- แสดงสรุปยอดเงินที่ได้รับ

### 4. **Story Section**
- แสดง Story ขอบคุณและความคืบหน้า
- แสดงผลลัพธ์ที่เกิดขึ้น

## API Response เมื่อครบจำนวน

เมื่ออนุมัติสลิปที่ทำให้ครบจำนวน:
```json
{
  "success": true,
  "message": "อนุมัติสลิปแล้ว และคำขอบริจาคครบจำนวนเป้าหมายแล้ว! 🎉",
  "is_completed": true,
  "completed_at": "2026-01-21T10:30:00.000000Z"
}
```

## การทดสอบ

1. สร้างคำขอที่มี goal_amount = 1000
2. ส่งสลิป 500 บาท → อนุมัติ → current_amount = 500
3. ส่งสลิป 500 บาท → อนุมัติ → current_amount = 1000, status = COMPLETED
4. ส่งสลิปเพิ่ม → ควรถูกปฏิเสธ

## หมายเหตุ

- ระบบจะตรวจสอบทุกครั้งที่อนุมัติสลิป
- ถ้ามีสลิปรออนุมัติหลายสลิป อาจทำให้เกินเป้าได้
- ควรแจ้งเตือน Organizer ให้ตรวจสอบสลิปที่รออนุมัติก่อน
